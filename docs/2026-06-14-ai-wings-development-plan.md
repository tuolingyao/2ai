# AI之翼 开发计划

> 本文档将验收合同的9个 Phase 拆分为可执行的开发任务。
> 验收口径以 `2026-06-14-ai-wings-acceptance-contracts.md` 为准，技术方案以 `2026-06-14-ai-wings-tech-architecture.md` 为准。

## Phase 1 — 产品口径、信息架构与统一内容模型

**目标**：项目初始化、数据模型落地、基础架构搭建。

| # | 任务 | 交付物 |
|---|---|---|
| 1.1 | 初始化 Next.js 项目（App Router、TypeScript、Tailwind、ESLint、Prettier） | 可运行的空项目 |
| 1.2 | 配置 Prisma + PostgreSQL，编写完整 Schema 并执行首次迁移 | 数据库表结构就绪 |
| 1.3 | 配置 NextAuth.js，实现邮箱注册/登录和角色（User/Editor/Reviewer/Admin） | 认证可用 |
| 1.4 | 编写数据库 seed 脚本，插入测试用分类标签和1个草稿场景 | 开发数据可用 |
| 1.5 | 搭建前台和后台布局骨架（Header/Footer/Sidebar），配置路由结构 | 页面路由可访问 |
| 1.6 | 配置环境变量模板（.env.example），编写项目 README 启动说明 | 新人可启动项目 |

**验收**：lint + typecheck + build 通过；数据库表与产品设计文档8种内容模型对应；新增场景路径不需要新增路由。

---

## Phase 2 — 前台首页、场景库与搜索发现

**目标**：用户能从首页发现场景，通过搜索和筛选找到目标。

| # | 任务 | 交付物 |
|---|---|---|
| 2.1 | 首页：品牌区、推荐场景、场景入口、学习路径说明 | 首页完整 |
| 2.2 | 场景库页：场景卡片列表、分类筛选、排序 | 场景库可用 |
| 2.3 | 搜索功能：关键词搜索场景标题/摘要，支持按分类、能力标签、工具标签筛选 | 搜索可用 |
| 2.4 | SceneCard 组件：封面图、标题、摘要、适合人群、分类标签 | 复用组件 |
| 2.5 | SEO：首页和场景库页 title/description/结构化数据 | SEO 元信息可配置 |
| 2.6 | 响应式：桌面/平板/移动端适配 | 三端可用 |
| 2.7 | 补充 seed 数据：至少3个已发布场景 + 分类标签 | 搜索可验证 |

**验收**：首页按场景能力进入；搜索筛选结果正确；非已发布内容不可见；响应式和自动化检查通过。

---

## Phase 3 — 场景页与统一四阶段学习路径

**目标**：场景页复用统一模板，四阶段结构完整展示。

| # | 任务 | 交付物 |
|---|---|---|
| 3.1 | 场景页顶部：场景定义、适合/不适合人群 | 场景入口信息完整 |
| 3.2 | StageTimeline 组件：四阶段总览时间线 | 路径全貌可见 |
| 3.3 | StageSection 组件：单阶段内容（能力标准、学习重点、练习任务、能力证据、AI 介入方式、工具建议） | 阶段内容完整 |
| 3.4 | ToolGuidanceCard 组件：当前工具/更优工具/迁移条件（后置展示） | 工具建议后置 |
| 3.5 | NodeCard 组件：节点入口卡片，链接到学习节点页 | 节点可进入 |
| 3.6 | 场景页底部：能力结果与下一步 | 闭环完整 |
| 3.7 | 补充 seed 数据：公众号场景完整四阶段内容 | 重点场景可验证 |
| 3.8 | 响应式适配 | 三端可用 |

**验收**：多个场景由同一模板渲染；阶段字段完整；工具建议后置；节点入口正确。

## Phase 4 — 学习节点页、练习任务与能力证据

**目标**：节点页完整展示学习内容，登录用户可交互。

| # | 任务 | 交付物 |
|---|---|---|
| 4.1 | 节点页：目标、前置条件、关键概念、方法重点、练习任务、通关标准、常见错误 | 节点字段完整 |
| 4.2 | AiDialogue 组件：工具选择、提示词、追问、用户补充、AI 输出、检查清单 | AI 示范完整 |
| 4.3 | 用户收藏功能：收藏/取消收藏场景 | 收藏可用 |
| 4.4 | 节点完成标记：标记节点为已完成/进行中 | 进度可记录 |
| 4.5 | 能力证据提交：EvidenceForm 组件，保存/编辑/删除能力证据 | 证据可保存 |
| 4.6 | 下一步导航：完成后跳转下一节点 | 导航正确 |
| 4.7 | 未登录边界：公开内容未登录可读，交互操作提示登录 | 权限边界正确 |

**验收**：节点字段与 AI 示例完整；学习记录和证据可保存；导航正确；权限边界通过。

## Phase 5 — 管理后台、权限与内容 CRUD

**目标**：后台完整 CRUD，三类角色权限正确。

| # | 任务 | 交付物 |
|---|---|---|
| 5.1 | 后台仪表盘：内容数量、已发布/草稿数量、最近更新、待审核 | 仪表盘可用 |
| 5.2 | 场景 CRUD：新建、编辑、复制、删除、排序 | 场景管理可用 |
| 5.3 | 阶段 CRUD：维护四阶段内容，关联场景 | 阶段管理可用 |
| 5.4 | 学习节点 CRUD：新建、编辑、排序、关联阶段、配置下一步 | 节点管理可用 |
| 5.5 | AI 对话示范 CRUD：关联节点，维护完整对话链路 | 对话示范可用 |
| 5.6 | 工具建议 CRUD：关联阶段或节点 | 工具建议可用 |
| 5.7 | 分类标签 CRUD：管理分类、目标人群、能力标签、工具标签 | 标签管理可用 |
| 5.8 | 媒体管理：上传/删除/关联场景或节点 | 媒体可用 |
| 5.9 | 角色权限：Admin 全权限、Editor 编辑提交、Reviewer 审核发布 | 三类角色权限正确 |
| 5.10 | 后台路由守卫：未授权访问重定向 | 安全边界通过 |

**验收**：三类角色权限正确；CRUD 全链路可用；前台只展示已发布内容。

---

## Phase 6 — 发布工作流、内容质量检查与 SEO

**目标**：发布状态流转完整，质量检查自动执行，SEO 可配置。

| # | 任务 | 交付物 |
|---|---|---|
| 6.1 | 发布状态流转：草稿 → 预览 → 待审核 → 已发布 → 已下线 | 状态机完整 |
| 6.2 | 预览功能：生成前台预览链接，仅后台可访问 | 预览可用 |
| 6.3 | 质量检查 API：检查四阶段完整、阶段字段完整、节点可导航、AI 示例完整、slug/SEO/封面/排序 | 检查可用 |
| 6.4 | 质量检查 UI：QualityCheckPanel 展示检查结果，不完整时阻止发布 | 阻断生效 |
| 6.5 | SEO 管理：场景/节点页 title/description/slug 可配置 | SEO 可配 |
| 6.6 | 结构化数据：场景页输出 JSON-LD | 搜索引擎可解析 |
| 6.7 | 发布/下线操作：发布后前台可见，下线后不可访问 | 状态边界正确 |

**验收**：不完整内容无法发布；完整内容可完成全状态流转；公开 SEO 正确。

---

## Phase 7 — 内容完整性、真实案例与可扩展运营

**目标**：重点场景内容真实完整，新增场景无需改代码。

| # | 任务 | 交付物 |
|---|---|---|
| 7.1 | 公众号场景内容完善：四阶段全部节点、真实 AI 对话示范、完整工具建议 | 重点场景可练 |
| 7.2 | 第二个示例场景：通过后台创建并发布一个完整场景（如"用 AI 做数据分析"） | 多场景验证 |
| 7.3 | 内容编辑流程验证：用 Editor 账号新建场景 → 填充内容 → 提交审核 → 发布 | 工作流验证 |
| 7.4 | 新增场景无代码验证：确认新增场景不需要新增组件、路由或数据结构 | 可扩展验证 |

**验收**：重点场景真实可练；新增场景无需代码上线；统一模型复用。

## Phase 8 — 账户、学习记录与前后台数据闭环

**目标**：用户学习数据完整，后台可查看基础统计。

| # | 任务 | 交付物 |
|---|---|---|
| 8.1 | 账户页：个人信息、收藏列表、最近学习、学习进度总览 | 账户页可用 |
| 8.2 | 继续学习：从账户页跳转到上次学习的节点 | 继续学习可用 |
| 8.3 | 阶段完成状态：四阶段全部节点完成后标记阶段完成 | 阶段进度可见 |
| 8.4 | 后台学习统计：用户数、节点完成数、活跃用户、热门场景 | 基础数据可查 |
| 8.5 | 数据隔离：用户只能访问自己的学习记录和证据 | 隐私正确 |

**验收**：账户页与学习记录可用；数据隔离正确；后台基础数据可查看。

## Phase 9 — 全链路回归、响应式与发布准备

**目标**：全链路验证通过，准备上线。

| # | 任务 | 交付物 |
|---|---|---|
| 9.1 | 前台全链路走查：首页 → 场景库 → 场景页 → 节点页 → 练习 → 证据 → 下一步 | 前台链路通畅 |
| 9.2 | 后台全链路走查：新建场景 → 编辑内容 → 预览 → 提交审核 → 发布 → 下线 | 后台链路通畅 |
| 9.3 | 权限回归：三类角色分别执行授权和未授权操作 | 权限无遗漏 |
| 9.4 | 响应式回归：桌面/平板/移动端全页面验证 | 三端可用 |
| 9.5 | SEO 回归：公开页面 title/description/JSON-LD/可抓取 | SEO 完整 |
| 9.6 | 性能检查：首屏加载时间、Lighthouse 评分 | 性能达标 |
| 9.7 | 安全检查：HTTPS、鉴权、防注入、数据隔离 | 安全达标 |
| 9.8 | Production build + 部署到 Vercel | 线上可访问 |
| 9.9 | 缺口清单：记录遗留问题与决策 | 清单为空或有明确非阻断决策 |

**验收**：自动化检查通过；浏览器验证通过；响应式和 SEO 验证通过；缺口清单为空或有明确决策。
