# AI之翼 Acceptance Contracts

本文件只定义各阶段 done 口径、回归边界、验证方式与证据；产品范围以 `2026-06-14-ai-wings-product-design-v2.md` 为准，实施顺序以 `C:\Users\PC\.claude\plans\steady-rolling-glacier.md` 为准。

## 通用验收规则

| 项 | Done 口径 |
|---|---|
| 产品一致性 | 实现不得偏离产品设计文档、项目指令和开发计划。 |
| 内容边界 | 公开入口只展示已发布内容；草稿、预览、待审核、已下线内容不得进入公开搜索、推荐或普通访问。 |
| 统一模型 | 新增场景、阶段、节点、案例、工具建议和标签必须走统一内容模型与后台维护能力。 |
| 学习主线 | 页面组织必须服务学习路径、能力标准、最小练习和能力证据；工具建议后置。 |
| 质量门槛 | 不得用占位案例、说明书式样板或硬编码专属页面凑数。 |
| 基础验证 | 每个实现阶段至少通过 lint、类型检查、production build；涉及 UI 的阶段必须做浏览器验证。 |

## Phase 1 — 产品口径、信息架构与统一内容模型

| Field | Contract |
|---|---|
| Done | 信息架构覆盖产品设计文档列出的前台与后台页面；内容模型覆盖 Scene、Stage、LearningNode、AiDialogueExample、ToolGuidance、Taxonomy、MediaAsset、SiteSetting 及用户学习相关记录；发布状态、排序、推荐、SEO、预览、下线和权限字段明确。 |
| Regression | 四份核心文档职责不互相覆盖；不得残留临时样板交付口径；工具建议仍从属于学习阶段。 |
| Verification | 核对产品设计、项目指令、项目索引、开发计划和模型定义；确认新增场景无需新增路由或页面结构；运行 lint、类型检查和 build。 |
| Evidence | 文档口径一致；模型能解释全部核心页面与后台模块；新增场景路径不改代码结构；自动化检查通过。 |

## Phase 2 — 前台首页、场景库与搜索发现

| Field | Contract |
|---|---|
| Done | 首页、场景库、分类/搜索/筛选/推荐/排序、场景卡片和 SEO 元信息可用；搜索覆盖场景、关键词、能力目标和工具名称。 |
| Regression | 首页仍按场景能力进入；搜索、推荐和排序不暴露非已发布内容；公开阅读不要求登录。 |
| Verification | 用桌面、平板、移动尺寸访问首页和场景库；执行关键词、分类、能力目标、工具名称搜索；检查不同发布状态可见性；运行 lint 和 build。 |
| Evidence | 入口与发现链路可用；搜索筛选结果正确；非已发布内容不可见；响应式与自动化检查通过。 |

## Phase 3 — 场景页与统一四阶段学习路径

| Field | Contract |
|---|---|
| Done | 所有场景页复用同一模板；正文一级结构为场景定义、四阶段总览、理解、上手、独立稳定产出、持续进阶、能力结果与下一步；阶段字段完整；重点公众号场景覆盖完整 AI 运营学习链路。 |
| Regression | 不为单个场景创建专属架构；阶段内不把工具建议置于能力标准之前；页面结构不依赖硬编码 slug。 |
| Verification | 打开重点场景和至少两个其他场景；逐项核对四阶段字段、工具后置和节点入口；检查主要响应式尺寸；运行 lint 和 build。 |
| Evidence | 多个场景由同一模板渲染；公众号场景展示具体 AI 操作过程；阶段字段完整；节点入口正确；自动化检查通过。 |

## Phase 4 — 学习节点页、练习任务与能力证据

| Field | Contract |
|---|---|
| Done | 节点页字段完整；AI 对话示范包含工具选择、提示词、追问轮次、用户补充、AI 输出和检查清单；登录用户可收藏、标记完成、保存能力证据并继续下一步。 |
| Regression | 单个节点只承担一个学习动作；能力证据服务能力标准；公开内容未登录可读；下一步不指向未发布节点。 |
| Verification | 访问多个节点页；完成一次练习记录、能力证据保存、节点完成和下一步跳转；检查未登录/登录边界；运行交互测试、lint 和 build。 |
| Evidence | 节点字段与 AI 示例完整；学习记录和能力证据可保存；导航正确；权限边界和自动化检查通过。 |

## Phase 5 — 管理后台、权限与内容 CRUD

| Field | Contract |
|---|---|
| Done | 后台登录、退出、角色权限、仪表盘、场景/阶段/节点/案例/工具建议/分类标签/媒体/SEO/站点设置 CRUD、预览、发布、下线和排序可用。 |
| Regression | 权限不能只靠前端隐藏；草稿编辑不影响公开页；删除或下线不能破坏引用内容且必须可审计或被阻止。 |
| Verification | 分别用管理员、内容编辑、审核发布者执行授权与未授权操作；完整创建、编辑、预览、发布、排序、下线一个场景及节点；上传并关联媒体；运行权限、CRUD、lint 和 build。 |
| Evidence | 三类角色权限正确；后台 CRUD 与发布操作可用；前台只展示已发布内容；自动化检查通过。 |

## Phase 6 — 发布工作流、内容质量检查与 SEO

| Field | Contract |
|---|---|
| Done | 草稿、预览、待审核、已发布、已下线状态流转可用；预览不进入公开入口；发布前检查四阶段、阶段字段、节点、AI 示例、slug、SEO、封面和排序；公开页面读取可配置 SEO。 |
| Regression | 不完整内容不能绕过后端质量检查发布；已下线内容不可公开访问；预览权限不泄露后台编辑能力；SEO 修改不破坏 slug。 |
| Verification | 用不完整场景发布并确认被阻止；补齐字段后预览、提交审核、发布、下线和重新发布；检查 SEO 与结构化信息；运行相关测试、lint 和 build。 |
| Evidence | 不完整内容无法发布；完整内容可完成全状态流转；公开 SEO 正确；搜索推荐只含已发布内容；自动化检查通过。 |

## Phase 7 — 内容完整性、真实案例与可扩展运营

| Field | Contract |
|---|---|
| Done | 重点公众号场景内容真实完整；关键节点展示具体 AI 使用过程；每阶段包含能力标准、练习任务、能力证据、常见未达标原因和补强动作；至少两个示例场景通过同一后台和模板发布。 |
| Regression | 不回到代做运营、工具排行榜、占位案例或硬编码示例模块；新增场景不要求新增组件、路由或专属数据结构。 |
| Verification | 逐页审查重点场景、示例场景和新增测试场景；用内容编辑账号新增并发布一个完整测试场景；核对节点 AI 对话链路；运行 lint 和 build。 |
| Evidence | 重点场景真实可练；新增测试场景无需代码上线；示例场景复用统一模型；内容质量检查和自动化检查通过。 |

## Phase 8 — 账户、学习记录与前后台数据闭环

| Field | Contract |
|---|---|
| Done | 用户登录、退出、账户页、收藏、最近学习、节点/阶段完成状态、能力证据增删改查和后台基础学习行为统计可用。 |
| Regression | 登录不阻断公开阅读；用户只能访问自己的学习记录；后台数据不泄露隐私；账户页不改变场景页和节点页结构。 |
| Verification | 用至少两个用户账号分别完成收藏、继续学习、节点完成和能力证据保存；确认数据隔离；在后台查看基础数据；运行认证、授权、lint 和 build。 |
| Evidence | 账户页与学习记录可用；能力证据正确保存；用户数据隔离正确；后台基础数据可查看；自动化检查通过。 |

## Phase 9 — 全链路回归、响应式与发布准备

| Field | Contract |
|---|---|
| Done | 前台学习链路、后台发布链路、账户学习记录、搜索筛选推荐排序、权限边界、响应式、SEO 和结构化信息均通过发布前验证。 |
| Regression | 修补不得破坏产品定位、统一四阶段、工具后置、发布边界、用户数据隔离、SEO 或后台维护路径。 |
| Verification | 运行完整 lint、类型检查、production build 和核心测试；启动应用，用浏览器走查前台、后台、账户和发布链路；检查桌面、平板、移动端；审查遗留缺口。 |
| Evidence | 自动化检查通过；浏览器验证通过；响应式和 SEO 验证通过；缺口清单为空或有明确非阻断决策。 |
