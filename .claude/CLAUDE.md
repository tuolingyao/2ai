## 多对话协作模式（必须遵守）

### 角色定义

| 对话角色 | 职责 | 禁止行为 |
|---|---|---|
| **计划/审查对话** | 定任务、写任务卡、审查代码、输出审查结论、推送部署 | **不直接改代码** |
| **执行对话** | 按任务卡开发、实现功能、提交代码 | 不自行变更需求范围 |
| **审查对话** | 对执行对话产出做二次审查 | — |
| **修 Bug 对话** | 处理测试失败、线上问题 | — |

### 核心约束

1. **串行流水线**：同一时间只有一个对话在改代码
2. **契约优先**：开发前必须有明确的任务卡（验收标准 + 检查点）
3. **审查闭环**：执行完成后必须回到计划/审查对话验收，不通过则输出修复任务卡

### 标准流程

```
1. 用户提出需求 → 计划对话调用 Skill 制定方案
2. 输出任务卡（目标、范围、验收标准、验证命令）
3. 执行对话按任务卡开发
4. 执行完成后 → 计划对话审查（构建/代码/业务/标准）
5. 发现问题 → 输出修复任务卡
6. 执行对话修复 → 再次审查
7. 通过后 → 计划对话推送代码、触发部署
8. 线上验证 → 关闭本轮迭代
```

### 任务卡模板

```markdown
## 任务：[任务名称]

### 目标
[一句话描述]

### 范围
- [ ] [具体项1]
- [ ] [具体项2]

### 验收标准
- [ ] 构建通过：`pnpm build`
- [ ] 无固定色残留：`rg "bg-white|text-zinc" src/app/admin`
- [ ] 业务逻辑不变：认证/权限/发布流程未破坏
- [ ] 视觉达标：暗夜模式无白块、无低对比文字

### 验证命令
```bash
pnpm build
```

### 禁止
- 不改 API、数据库、权限、发布逻辑
- 不重构未要求的内容
```

### 审查清单

| 检查项 | 标准 |
|---|---|
| 构建 | `pnpm build` 通过 |
| 固定色 | 后台无 `bg-white`、`text-zinc-*`、`bg-zinc-*` 残留 |
| 业务逻辑 | 认证、权限、发布状态、数据隔离未破坏 |
| 主题适配 | 暗夜模式无大面积白块、无低对比文字 |
| 代码风格 | 匹配项目现有风格，无新增注释（除非用户要求） |

## 技术规范速查

### 错误码体系
- 用户 10xxx, 参数 20xxx, 词库 30xxx, 学习 40xxx
- 卡牌 50xxx, 战斗 60xxx, 炼丹 70xxx, 渡劫 80xxx
- 宗门 90xxx, 支付 A0xxx, 防沉迷 B0xxx, 系统 C0xxx

### API 命名
- URL: snake_case `/api/v1/word_roots/{id}`
- 字段: camelCase `userName`, `createdAt`
- 白名单: 注册/登录/刷新Token/健康检查/公开词库

- 后端单元 ≥ 90%, 前端单元 ≥ 80%, 领域层 100%, 契约测试 100%
